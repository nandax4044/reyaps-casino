TROUBLESHOOTING
═══════════════════════════════════════════════════════════════

PROBLEM: SQL error "relation users does not exist"
SOLUTION:
  - Table users mungkin di schema auth, bukan public
  - Di SQL script, ganti semua:
    REFERENCES users(id) 
  - Dengan:
    REFERENCES auth.users(id)
  - Run SQL lagi

PROBLEM: Bait masih 0 setelah grant
SOLUTION:
  1. Buka browser DevTools (F12)
  2. Tab: Network
  3. Grant bait lagi
  4. Cari request: grant-bait
  5. Check response:
     - Jika success: true → Backend OK
     - Check inventory.bait atau inventory.bait_balance
  6. Jika bait ada di response tapi tidak muncul di UI:
     - Refresh page (Ctrl+F5)
     - Clear cache
  7. Jika masih 0, screenshot response dan kasih tau saya

PROBLEM: "Endpoint not found" saat pilih user/rod
SOLUTION:
  - Error ini harusnya sudah fix setelah deploy
  - Check Console (F12 → Console tab)
  - Screenshot error dan kasih tau saya

PROBLEM: SQL sudah run, deploy sudah, tapi masih error
SOLUTION:
  1. Verify tables di Supabase:
     - Table Editor (sidebar)
     - Check ada: fishing_inventory, user_rods, etc.
  2. Check Vercel logs:
     - vercel.com/dashboard
     - Deployment logs
     - Function logs
  3. Check browser console logs saat grant bait

TABLES YANG DIBUAT
═══════════════════════════════════════════════════════════════

1. fishing_access
   - Tracks user fishing access permissions
   - Columns: user_id, granted_by, expires_at, is_active

2. user_rods
   - Tracks rod ownership per user
   - Columns: user_id, rod_id, granted_by, notes

3. fishing_inventory ⭐ (PENTING UNTUK BAIT!)
   - Stores user's bait and fishing saldo
   - Columns: user_id, bait, fishing_saldo
   - UNIQUE constraint on user_id

4. fish_inventory
   - Logs all fish caught
   - Columns: user_id, fish_name, fish_lb, fish_value

5. afk_fishing_sessions
   - Persistent AFK sessions
   - Columns: user_id, username, equipped_rod, is_active
